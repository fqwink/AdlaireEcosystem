# Adlaire Ecosystem

## コア原則

| 原則 | 詳細 |
|------|------|
| **単一ファイル原則** | `Core.php` 1ファイルに全実装を収める |
| **外部依存ゼロ** | Composer不要、サードパーティライブラリ一切なし |
| **PHP 8.3以降** | 起動時にバージョンチェック、8.2以前は即時エラー終了 |
| **フロントエンド機能なし** | フロントエンド系の機能は一切実装しない |
| **段階的拡張** | 確定仕様をベースに改良・新規機能を追加していく |

---

## モジュール仕様

### 1. ルーティング

| 項目 | 詳細 |
|------|------|
| HTTPメソッド | GET / POST / PUT / PATCH / DELETE |
| クエリパラメータ | 取得対応 |
| ルートグループ | プレフィックス対応 |
| エラーハンドリング | 404 / 405 |

### 2. リクエスト

| 項目 | 詳細 |
|------|------|
| メソッド取得 | `GET` `POST` `PUT` `PATCH` `DELETE` |
| URI取得 | リクエストURIの取得 |
| ヘッダー取得 | 任意のヘッダー取得 |
| ボディパース | JSON のみ |
| クエリパラメータ | 取得対応 |
| IPアドレス | クライアントIP取得 |

### 3. レスポンス

| 項目 | 詳細 |
|------|------|
| ステータスコード | 任意のHTTPステータスコード設定 |
| ヘッダー設定 | 任意のレスポンスヘッダー設定 |
| JSON出力 | JSONレスポンスの構築・送出 |

### 4. バリデーション

| ルール | 詳細 |
|--------|------|
| `required` | 必須チェック |
| 型チェック | `string` / `int` / `float` / `bool` / `array` |
| `min` / `max` | 最小・最大値チェック |
| `regex` | 正規表現チェック |
| `email` | メールアドレス形式チェック |
| エラー構造化 | バリデーションエラーの構造化返却 |

## ファイル構成

```
Core.php       # フレームワーク本体（単一ファイル）
app.php        # アプリエントリポイント
```
